## 1. Overview of Enumeration

Enumeration is the **first phase of active exploitation**. While scanning identifies that a "door" is open, enumeration is the process of actively reaching through that door to see who is inside, what they are doing, and what they have in their pockets.

### üìç What is Enumeration?
Enumeration is defined as the process of extracting detailed information from a target system using active connections and directed queries. It is a **systematic probing** of a target to identify valid system accounts, network resources, shares, and services.

> üí° **Analogy:** If *scanning* is checking which windows in a house are unlocked, *enumeration* is **opening the window** and looking at the labels on the file folders sitting on the desk inside.

## 2. Information Gathered during Enumeration

Enumeration aims to collect specific, actionable intelligence that can be used to plan a direct attack:

- **Network Resources & Shares:** Identifying shared folders, printers, and network drives.
- **Users and Groups:** Extracting valid usernames, RID (Relative Identifiers), and group memberships.
- **Routing Tables:** Understanding how the network is structured internally.
- **Audit & Service Settings:** Discovering password policies, account lockout thresholds, and running service details.
- **Machine Names:** Identifying the hostnames and roles of specific servers.
- **Applications and Banners:** Grabbing detailed version info for web servers, databases, and middleware.
- **SNMP & DNS Details:** Extracting device-specific configuration data and zone records.

## 3. Services and Ports to Enumerate (The "Where")

The following table lists the critical services and ports that ethical hackers target during the enumeration phase.

| Service | Port(s) | Protocol | Why We Enumerate It |
| :--- | :--- | :--- | :--- |
| **DNS Zone Transfer** | 53 | TCP | To replicate entire DNS databases and map internal hosts. |
| **Microsoft RPC** | 135 | TCP/UDP | To map RPC services and endpoint locations. |
| **NetBIOS Name Svc** | 137 | UDP | To resolve machine names and identify domain roles. |
| **NetBIOS Session** | 139 | TCP | To access shares, list users, and perform Null Session attacks. |
| **SNMP** | 161 | UDP | To extract device info, system uptimes, and user lists (via community strings). |
| **SNMP Trap** | 162 | TCP/UDP | To listen for unsolicited notifications from network agents. |
| **LDAP** | 389 | TCP/UDP | To query Active Directory for users, groups, and device objects. |
| **SMB over TCP** | 445 | TCP | To perform direct-host share enumeration and exploit SMB vulnerabilities. |
| **IKE (IPSec)** | 500 | UDP | To discover VPN gateways and identify encryption parameters. |
| **SMTP** | 25 | TCP | To enumerate valid email users via VRFY, EXPN, and RCPT commands. |
| **NFS** | 2049 | TCP | To identify exported file systems and unprotected network storage. |
| **Global Catalog** | 3268 | TCP | To search for forest-wide objects in a multi-domain AD environment. |

## 4. Enumeration Techniques & Types

Enumeration is often categorized by the protocol or service being targeted.

### üîπ NetBIOS Enumeration
Uses the NetBIOS (Network Basic Input/Output System) to discover hostnames, shares, and user info. 
- **Key Tools:** `nbtstat`, `enum4linux`, `nmap --script nbstat.nse`.

### üîπ SNMP Enumeration
Queries the Simple Network Management Protocol to gather device-level data. The "Read" community string (often `public` by default) is the primary target.
- **Key Tools:** `snmpcheck`, `snmp-check`, `onesixtyone`.

### üîπ LDAP Enumeration
Queries the Lightweight Directory Access Protocol to extract massive amounts of data from Active Directory.
- **Key Tools:** `ldapsearch`, `ADExplorer`, `Softerra LDAP Browser`.

## 5. CEH Exam Traps & Tips

| ‚ö†Ô∏è Common Trap | ‚úÖ Correct Understanding |
| :--- | :--- |
| **Thinking Enumeration is Passive** | Enumeration is **ACTIVE**. It involves creating direct connections to the target. |
| **Confusing SNMP Versions** | SNMP v1 and v2 use weak community strings; **SNMP v3** is the only one with encryption/auth. |
| **Null Session vs. SMB** | A Null Session is a connection with no username/password. It usually targets port 139/445. |
| **SMTP Enumeration Commands** | Know these: `VRFY` (verify user), `EXPN` (expand list), `RCPT TO` (identify recipients). |
| **RID (Relative Identifier)** | RIDs are appended to the SID. User 500 is always the built-in Administrator account. |
